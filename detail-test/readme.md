# datails 要素のスタイルテスト

2022.01.10
現状のdetails要素がどこまでカスタマイズできるかの実験

# 実験内容

## 特殊なスタイルを指定しない場合の挙動

ページには`<summary>`タグの内容だけが表示される。
クリックすると`<details>`タグの全容が表示される。その際、details要素にopen属性が付与される（open=>true）

close時はsummary要素以外の内容は`height: 0`で計算され、open時に`height: auto`のような扱いになる模様。

## summaryと本文の表示場所を分離したい

summary要素を`position: absolute`や`fixed`にするとsummaryの内容をdetailsから分離できる。
この方法で分離するとdetails要素が閉じている状態でのdetails要素自体の高さは0になるので、適切なマージンをdetails要素に付与しておかないと前後の要素と重なってしまう。

本文の要素のみを画面全体に表示するようなこともできるが、summary要素をクリックしないと本文を閉じることができないのでsummary要素が隠れるような表示方法にはできない。
バックドロップになるような背景をつけ、バックドロップをクリックするとdetailsのopen属性をfalseにするなどのスクリプトを付与すれば本文を閉じることができる。
そのようなことをするくらいなら初めからdetailsを使わない折り畳みの方法を実装したほうが良いと思われる。
（現段階ではdetailsの開閉にアニメーションやトランジションは効果がない）
